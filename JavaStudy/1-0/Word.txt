OS

　OS
　　Operating System
　　基本ソフトウェア
　　ユーザーやアプリに対してインタフェースを提供し
　　ハードウェアに対しては各リソースの管理を行なっている

　Windows
　　デスクトップOSとしては88％のシェアを誇る
　　最新のWindows10の普及とともに過去のWindos7は減少傾向にある
　　多くのソフトウェアとの相互性があるので便利だが脆弱性がある
　　多くのハードウェアがある。

　MacOS
　　GUIを導入していて直感的に操作できる
　　他のOSをインストールできる
　　プリインストールされたソフトウェアが豊富
　　Apple製品同士との連携力が高い

　Unix
　　現存する中で最も古いOS
　　UnixをもとにMacOSができた
　　使用割合は低いが企業内でのサーバー、ワークステーション上で稼働している

　Linux
　　Unixとよく似ている（Unix系）
　　オープンソース
　　LinuxをもとにAndroidができた

SQL

　SQL（シーケル）
　　データベース言語
　　データベースにデータを挿入、管理、検索を行う

　　DB2
　　　他と比べシンプル
　　　デッドロックが起きやすい
　　　コストが低い

　　MySQL
　　　オープンソース
　　　大きなサイトの利用に適している
　　　高速検索可能なのでGoogle,Youtubeなどが採用している

　　　Maria DB
　　　　MySQLの派生で生まれた
　　　　WIkipediaやBooking.comが有名
　　　　バグや開発計画に関する情報、修正したバグに対するテストケースも常に公開されている

　　Oracle DB
　　　コストは高いがサポートは充実
　　　複雑な処理も可能
　　　堅牢性が高い

　　PostgreSQL
　　　オープンソース
　　　コストが低い
　　　パラレルクエリなので処理が早い

　　SQLite
　　　設定不用の自己完結型のシステムなので依存度が低く多くのOSに対応
　　　軽量コンパクト
　　　中小規模の開発向き

　　SQL Server
　　　GUIの利用可能で直感的に操作出来る
　　　規模に合わせたエディションを選択できる
　　　マイクロソフトの開発言語と相性がいい（C#、Windows）

　CRUD
　　Create,Read,Update,Delete,データを操作する最低限の機能
　　生成、読み取り、更新、削除
　　ソフトの基本

　トランザクション
　　複数の処理を一つにまとめたもの
　　
　　

　　コミット
　　　処理や変更を確定させる
　　　トランザクションの処理の成功したのち結果を確定させること
　　　ファイルをレポジトリへ登録すること

　　ロールバック
　　　何か問題が起きた時にそれまでの処理を取り消し
　　　前の状態へ戻すこと
　　　コミットされてないトランザクションの処理を取り消し戻すこと

　排他制御
　　データやファイルに対して複数のアクセスが見込まれる場合
　　あるトランザクションがアクセス中の場合
　　他のトランザクションのアクセスを拒否し不整合を防ぐ

　　楽観ロック
　　　滅多に同時更新は起きないであろうと考え
　　　データ変更後更新対象のデータがデータ取得時と同じであるか確認してから
　　　更新することで整合性を保証する方式

　　悲観ロック
　　　よく同時更新が起こるであろうと考え
　　　データを取得する際にロックをかけトランザクション開始直後から
　　　コミット、ロールバックされるまで他のトランザクションから更新されないことで整合性を保つ

　実行計画
　　データベースがSQL文を実行できるようにオプティマイザが
　　実行計画を立案し１ステップずつ実行していく
　　

　チューニング
　　主に設定変更などを通じ使用状況へ適した状態に調整すること
　　データベースではデータの入れ方取り出し方、箱（ディスクなど）を変更し
　　より負荷を分散させたりする。

　パフォーマンス
　　性能がいい、性能が悪いの言い換えで使う
　　頑張り具合
　　

　スロークエリ
　　遅いSQLのこと
　　
　　

　クエリログ
　　クライアントからの接続解除情報
　　実行されたSQLクエリを出力してくれるログ
　　エラーが出た時に役立つ

肩書

　PM
　　プロジェクトマネージャー
　　プロジェクトの企画、予算、人材確保など準備、調達を行う
　　クライアントを交えて企画を行い上記のことや納期を取り決める
　　プロジェクト終了後は報告書作成を行う
　　メインは、マネジメント業務だが開発業務に参加する場合もある。

　PL
　　プロジェクトリーダー
　　中大規模の開発になるとPMの代わりにプロジェクトチームの管理、統括をする役割
　　サブチームのリーダーとして進歩、品質などを管理する
　　PMは基本一人だがPLは複数の存在

　PG(PE)
　　プログラマー
　　SEが作成した仕様書をもとに実際にコードを書いていく
　　テストを行いエラーがないか確認する
　　近年PGの人件費が安い海外に業務を発注するオフショア開発も盛ん

　SE
　　システムエンジニア
　　要件定義、仕様書作成、コーディング、テスト、開発案件全体
　　SE兼PGとなることも多い
　　PMやPLの受け身にならず能動的に動くことが求められる

　デザイナー
　　主に製品のレイアウトを考える業務
　　製品のコンセプトやターゲット層に合わせたデザインを考慮していく
　　

　エンジニア
　　工学（エンジニアリング）に関する知識やスキルを使った仕事をしている人
　　技術者とも言われる
　　ITエンジニアとしても使われる

　　フロントエンド・エンジニア
　　　クライアント側で作動する部分を作るエンジニア
　　　Webアプリのユーザーが画面越しに触れる部分を作る
　　　現場によってデザインに触れる場合もある

　　サーバーサイド・エンジニア
　　　サーバー側でデータ処理環境の構築・保守点検業務する人
　　　基幹系企業で使われる

各種プロコトル

　HTTP
　　Hyper Text Transfer Protocol
　　WWWでサーバーとクライアントがHTMLで書かれた文書などの情報をやりとりする時に使われるプロコトル
　　

　FTP
　　File Transfer Protocol
　　ファイル転送プロトコル
　　サーバーにアップされたファイルの一覧をみることやファイルをサーバーにアップロードしたり、ダウンロードしたりする

　HTTPS
　　HTTPとの違いは通信内容を暗号化されているか否かの違い
　　より安全にWeb情報のやりとりをすることができる
　　

　SSL
　　Secure Sockets Layer
　　Webブラウザとウェブサーバー間のデータの通信を暗号化し送受信させる仕組み
　　SSLにより暗号化されているデータをHTTPSとなる

　SSH
　　Secure Shell
　　ネットワークに接続された機器を遠隔操作し管理するための手段
　　通信途中の情報は暗号化される

　ポート、ポート番号
　　IPアドレス内のどこにあるか示す情報
　　
　　

　HTTP、FTP、SSHの各ポート番号
　　TCP 443 : HTTPS
　　TCP 20 : FTP (データ)TCP 21 : FTP (制御)
　　TCP 22 : SSH

　ファイアーウォール
　　企業などの内部ネットワークをインターネットを通して
　　侵入してくる不正なアクセスから守るための“防火壁”
　　

　TCP
　　通信手段の一つ（通信プロトコル）
　　安全性重視だが転送速度が低い
　　


IPアドレス

　IPアドレス
　　インターネット上に接続された機器が持つ番号
　　ネットワーク上で通信相手を間違わないようにするために使われる
　　

　　グローバルIPアドレス
　　　世界的な管理が行われる世界で重複しないアドレス
　　　
　　　

　　プライベートIPアドレス
　　　LANなどの企業内ネットワークに使われるアドレス
　　　他のネットワークでは同じアドレスが存在する可能性もある
　　　


ドメイン

　ドメイン
　　インターネット上の住所
　　Webサイトがどこにあるかを表している
　　

　DNS
　　Domain Name System
　　ドメインをインタネット上で管理するためのシステム
　　IPアドレスの数字をわかりやすいようにするための仕組み

　VPN
　　Virtual Private Network
　　VPNに接続しインターネットを使うとセキュリティに強い通信になる
　　

　ブリッジ
　　ネットワーク上のデータのMACアドレスを見て機器先を振り分ける役割
　　データ中継機器
　　

　プロキシサーバー
　　内部の外部のネットワークを高速や安全な通信を確保するための中継サーバ

　キャッシュ
